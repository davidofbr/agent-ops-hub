# `personas.md` template

Template for the personas file generated by `init-brand-context`. Personas
are **individual buyer archetypes**, not segments. This file is designed
so that downstream skills can spin up agents that embody these personas
for message testing, content review, or simulated sales calls.

Segment-level targeting criteria belong in `icp.md`, not here. The two
files are complementary and both should be present in a healthy repo.

## Rules

1. **2–4 personas only.** More dilutes the operational value. If the client
   has 5+ distinct buyer types, this is probably an ICP problem (too broad)
   or a product-line problem (separate personas per product) — flag it
   in the Phase 10 summary.

2. **Language & vocabulary must contain real quotes** where any exist in
   the scrape. Synthesized phrases are `[I]` and capped low.

3. **Search behavior is `[UNKNOWN]` by default** unless explicitly provided.
   Marketing copy rarely states this.

4. **Voice / tone notes are inferred.** Unless the user provides explicit
   guidance, these are `[I]`. Downstream agents should treat them as a
   starting point, not ground truth.

5. **Confidence cap without customer data.** Without call transcripts,
   win/loss interviews, or sales call recordings, the sources dimension
   caps at 60% because marketing copy alone is aspirational not validated.
   State this in the Phase 10 summary explicitly.

6. **The confidence header is the top two comment lines.** Consumer
   skills parse them on load. No content precedes them.

7. **Two personas minimum.** If the scrape + interview cannot produce
   two distinct archetypes, write one real persona plus one placeholder:

   ```
   ## Persona 2: [UNKNOWN]
   > `[UNKNOWN — only one persona identified at init. Secondary buyer type not yet characterized.]`
   ```

   Never invent a second persona to fill space.
